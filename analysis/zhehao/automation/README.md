	此项目是一个自动化绘图工具，能够自动遍历指定的 MC 信号样本与 Data 背景样本，在不同的数据目录中提取物理特征并生成带有动态自适应坐标轴的归一化分布图。（采用max-min绘图，因此可能出现图像有效部分被压缩等问题）
	在首次运行前，需要在主程序同一目录下配置一个名为 branch_config.py 的文件，并将所有需要分析的 Branch 变量名存入其中的 all_branches 列表。未来对branch的修改均在此中进行。注意绘图所需的mc文件与数据文件也应当保存在此文件夹中。这些root文件的命名与张志财老师Labserver服务器上提供的文件名完全一致
	在具体使用时，运行主程序后终端会弹出一个交互式提示，要求用户输入正则表达式（Regex）来筛选当前想要绘制的变量。例如可以输入 ^W_PT$ 来精确只画这一个变量进行快速测试，或者输入 _PT$ 来批量绘制所有横动量相关的特征；如果直接按回车，程序则会静默且带进度条地全量遍历配置文件中的所有变量。	
	生成的图片文件会被命名为形如5GeVCtau10ps_OS1J_nPVs.png 的格式，其含义是易见的。文件将默认保存在一个（新建的）文件夹Output_Plots中
	正则表达式的用法：（from Gemini 3.1 Pro）

	1. 核心符号速查
^ ：表示字符串的开头。
$ ：表示字符串的结尾。
. ：表示任意单个字符。
* ：表示前面的字符可以重复零次或多次（.* 连起来就代表“任意长度的任意字符”）。
| ：表示逻辑或（OR）。
() ：用于将多个条件括起来作为一个整体。
2. 常见使用场景与示例
场景一：只想画某一个特定的变量（精确匹配）
如果你只想测试或查看 nPVs 这个变量，为了防止匹配到名为 nPVs_weight 或 Old_nPVs 的其他变量，你需要使用 ^ 和 $ 把变量名“锁死”。

输入: ^nPVs$
效果: 严格且只匹配 nPVs。
场景二：想画某一类物理对象的所有变量（前缀匹配）
如果你正在研究一号喷注（Jet1），想要画出它的横动量、质量、eta 等所有相关变量，你可以锁定开头。

输入: ^Jet1_
效果: 匹配所有以 Jet1_ 开头的变量，如 Jet1_PT, Jet1_M, Jet1_ETA 等。
场景三：想画所有粒子的某一种运动学特征（后缀匹配）
如果你想对比所有粒子的横动量（PT），你可以锁定结尾。

输入: _PT$
效果: 匹配所有以 _PT 结尾的变量，如 W_PT, Mu_PT, Jet1_PT 等。
场景四：模糊搜索（包含匹配）
如果你记不清全名，只知道变量里包含某个关键字（比如你想看所有和 Primary Vertex 相关的变量）。

输入: PV
效果: 只要名字里带有 PV 的都会被画出来，如 nPVs, ALLPVX, PV_Z 等。
场景五：同时指定多个不同的变量（多条件组合）
如果你在写报告，今天只需要提取三个特定的变量放进 PPT，可以使用 | 将它们连接起来。

输入: ^(W_PT|W_M|nPVs)$
效果: 精确匹配这三个指定的变量，其他的全部忽略。(注意：外面的 ^ 和 $ 以及括号 () 是必须的，否则会变成模糊匹配)
场景六：我全都要（全量匹配）
如果你的配置文件 branch_config.py 里都是你需要的变量，你想一次性批量跑完去吃个饭。

输入: .* （或者直接什么都不输，按回车键）
效果: 匹配配置文件中的所有变量，自动机将满载运行。
